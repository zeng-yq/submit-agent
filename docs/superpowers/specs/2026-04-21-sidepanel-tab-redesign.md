# Sidepanel 三 Tab 重设计

## 背景

当前 sidepanel 使用全局 header + 视图切换模式。Header 中包含产品选择器和导航按钮，但外链分析和设置功能与产品无关，导致 UI 和使用逻辑不一致。

## 目标

将 sidepanel 按功能分为三个独立 Tab，产品选择器只在需要它的 Tab 中出现。

## 整体布局

```
┌─────────────────────────────────┐
│  外链提交  │  外链分析  │  设置   │  ← 顶部 Tab 栏（固定）
├─────────────────────────────────┤
│                                 │
│         当前 Tab 内容            │
│         （可滚动）               │
│                                 │
└─────────────────────────────────┘
```

- Tab 栏高度约 40px，固定在顶部
- 选中态用下划线或背景色高亮（主色调琥珀金）
- 内容区占满剩余高度
- 去掉现有的 header（产品选择器 + 导航按钮），由 Tab 栏替代

## Tab 1：外链提交（替代 Dashboard）

```
┌─────────────────────────────────┐
│  外链提交  │  外链分析  │  设置   │
├─────────────────────────────────┤
│  [产品A ▾]  [+ 添加产品]        │  ← 产品选择器
├─────────────────────────────────┤
│  进度统计条                      │
│  全部 | 已完成 | 失败 | 当前提交  │
│  [搜索...]                      │
│  SiteCard 列表                  │
│  活动日志（可折叠）              │
└─────────────────────────────────┘
```

- 产品选择器从全局 header 移入此 Tab 内部，功能和样式不变
- 下方保留现有 Dashboard 的全部内容：进度条、状态筛选、搜索、站点卡片、活动日志
- 无产品时直接显示 QuickCreate
- 产品选择器下拉菜单中保留「管理产品」链接（跳转 options 页面）

## Tab 2：外链分析

- 内容与现有 `BacklinkAnalysis` 组件一致
- CSV 导入、批量分析控制、外链表格、状态筛选、批次管理、活动日志
- 无产品选择器
- 数据是全局的，不随产品切换变化

## Tab 3：设置

- 内容与现有 `SettingsPanel` 组件一致
- 数据同步（Google Sheets）、AI 模型配置、偏好设置
- 无产品选择器

## 变更范围

### 需要修改的文件

1. **`extension/src/entrypoints/sidepanel/App.tsx`** — 主要改动：
   - 移除现有 header（产品选择器 + 导航按钮）
   - 添加 Tab 栏组件（三个 Tab）
   - 将产品选择器移入外链提交 Tab 内容区
   - 用 `useState` 管理当前 Tab 状态

2. **`extension/src/components/Dashboard.tsx`** — 接收产品选择器作为顶部内容，或由 App.tsx 直接在 Tab 内组合

### 不需要修改的文件

- `BacklinkAnalysis.tsx` — 内容不变
- `SettingsPanel.tsx` — 内容不变
- `QuickCreate.tsx` — 逻辑不变
- `options/App.tsx` — 不受影响

### 移除的元素

- sidepanel header 中的导航按钮（外链分析、产品管理、设置）— 由 Tab 栏替代
- 全局产品选择器 — 移入外链提交 Tab

## 实现约束

- 保持现有的暖色调石板色系风格
- Tab 栏在 sidepanel 窄宽度下仍可用（文字紧凑）
- 不引入新依赖
